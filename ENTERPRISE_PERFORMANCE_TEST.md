## Методология тестирования

### 1. Традиционный поиск (grep)
```bash
time grep -r "pattern" --include="*.xml" --include="*.bsl" --include="*.os" .
```

### 2. Векторный поиск (ChromaDB)
```bash
# Индексация
time python3 chroma_simple_server.py --index --project-root /path/to/project

# Поиск
time python3 chroma_simple_server.py --search "query" --results 10 --project-root /path/to/project
```

## Рекомендации для enterprise-проектов

### Когда использовать **grep**:

1. **Точный поиск** по известным ключевым словам
2. **Поиск имен** файлов, функций, переменных
3. **CI/CD пайплайны** где критична полная проверка
4. **Сложные регулярные выражения**
5. **Когда нужны ВСЕ совпадения**

### Когда использовать **Chroma Vector Search**:

1. **Семантический поиск** по смыслу и концепциям
2. **Обучение новых разработчиков** (поиск по описанию функциональности)
3. **Архитектурный анализ** (нахождение связанных компонентов)
4. **Абстрактные запросы** ("как работает модуль оплаты?")
5. **Когда важна релевантность, а не полнота**

### Гибридный подход для enterprise:

1. **Использовать ChromaDB для:**
   - Первоначального исследования кодовой базы
   - Семантического поиска по концепциям
   - Обучения и документации

2. **Использовать grep для:**
   - Точного рефакторинга
   - Поиска всех вхождений конкретного идентификатора
   - Автоматизированных проверок в CI/CD
