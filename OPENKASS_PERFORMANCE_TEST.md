## Методология тестирования

### 1. Традиционный поиск (grep)
```bash
# Замер времени выполнения
time grep -r "pattern" --include="*.java" .
```

### 2. Векторный поиск (ChromaDB)
```bash
# Индексация
time python3 chroma_simple_server.py --index --project-root ../openkass

# Поиск
time python3 chroma_simple_server.py --search "query" --results 5 --project-root ../openkass
```

## Сравнительный анализ

### Скорость (меньше = лучше)
```
grep:      0.004s - 0.021s   (в 300-2000 раз быстрее)
ChromaDB:  6.359s - 7.733s   (включая загрузку модели)
```

### Качество результатов
| Критерий | grep | ChromaDB |
|----------|------|----------|
| **Точность** | Высокая (точные совпадения) | Высокая (семантическая релевантность) |
| **Полнота** | Низкая (только ключевые слова) | Высокая (связанные концепции) |
| **Ранжирование** | Нет | Да (по релевантности) |
| **Контекст** | Только строка | Фрагмент кода с номером строки |
| **Естественный язык** | Не поддерживается | Поддерживается |

## Выводы

### Когда использовать **grep**:
1. **Мгновенный поиск** по известным ключевым словам
2. **Малые проекты** (< 50 файлов)
3. **Точные имена** файлов, функций, переменных
4. **CI/CD пайплайны** где критична скорость
5. **Сложные регулярные выражения**

### Когда использовать **Chroma Vector Search**:
1. **Семантический поиск** по смыслу, а не по словам
2. **Крупные проекты** (1000+ файлов)
3. **Обучение новых разработчиков** (поиск по концепциям)
4. **Нахождение связанного кода** (паттерны, архитектура)
5. **Кроссязычные проекты** (код + комментарии на разных языках)
6. **Абстрактные запросы** ("как работает авторизация?")

### Для проекта OpenKass:
- **grep оптимален**: 7 файлов, мгновенные результаты
- **ChromaDB избыточен** для такого объема, но демонстрирует:
  - Мощь семантического поиска
  - Возможность находить связанные концепции
  - Преимущество для сложных абстрактных запросов

## Рекомендации

### Для небольших проектов (< 100 файлов):
- Используйте **grep** для повседневных задач
- Рассмотрите **ChromaDB** только для:
  - Документации и обучения
  - Сложных архитектурных вопросов
  - Поиска по естественному языку

### Для enterprise-проектов (> 1000 файлов):
- **ChromaDB становится незаменимым**:
  - Поиск функциональности по описанию
  - Нахождение всех реализаций паттерна
  - Обнаружение связанного кода
  - Анализ архитектуры
